Design a production-grade, multi-tenant system architecture for TriFi / Ifyrt, a Telegram-native automated trading simulation platform with strict separation between simulation and live execution layers, emphasizing deterministic, high-fidelity backtesting as the core product. The platform operates on a subscription model with a 7-day free trial, followed by a $6.99/month paid tier, and includes an optional, subscription-gated dashboard that complements (but does not replace) full Telegram control. The platform also supports a simulation-based copy-trading system, enabling users to risk-free simulate the strategies of consenting peers.

Why TriFi / Ifyrt: The name is psychologically resonant (rhymes with Binance), easy to recall, and implies experimentation and trial (“Try”) with trading, reinforcing the platform’s educational focus.

## The n8n "Brain Stem" — How Everything Wires Together

n8n is self-hosted on Railway and acts as the **central nervous system**. Every service fires webhooks **into n8n**, and n8n routes, transforms, and triggers downstream actions.

```
Telegram command received
        │
        ▼
n8n Workflow: "Route Telegram Event"
        │
        ├──► /backtest   ──► trigger Simulation Engine worker
        ├──► /simulate   ──► trigger Real-Time Sim worker
        ├──► /live_on    ──► check subscription → enable Live Exec
        ├──► /copy       ──► trigger Copy-Trading Sim layer
        ├──► /subscribe  ──► trigger Subscription service
        └──► /status     ──► query Supabase → reply via Telegram
```

Every workflow in n8n:
- **Receives** a webhook or Supabase DB trigger
- **Validates** subscription/auth via Supabase query node
- **Fires** the correct Railway service
- **Writes results** back to Supabase
- **Notifies** user via Telegram Bot API call

## Core Event Payload Format

Every command produces a **standardized IfyrtEvent**. This is the contract between the bot and n8n — never changes shape.

```typescript
// types.ts
export interface IfyrtEvent {
  event_id: string;          // uuid
  event_type: EventType;     // enum below
  timestamp: string;         // ISO 8601
  source: "telegram";
  user: {
    telegram_id: number;
    username: string | null;
    first_name: string;
  };
  payload: Record<string, unknown>; // command-specific data
  raw_text: string;                 // original message
  chat_id: number;                  // for reply routing
  message_id: number;
}

export type EventType =
  | "user.start"
  | "user.help"
  | "backtest.request"
  | "simulation.start"
  | "simulation.stop"
  | "user.status"
  | "live.enable"
  | "live.disable"
  | "copy.start"
  | "copy.stop"
  | "subscription.view"
  | "dashboard.request"
  | "strategy.list"
  | "key.submit";
```

## index.ts — The Webhook Server

```typescript
import express from "express";
import { parseCommand } from "./parser";
import { dispatchToN8n } from "./dispatcher";
import { replyToUser } from "./replies";

const app = express();
app.use(express.json());

const TELEGRAM_TOKEN = process.env.TELEGRAM_BOT_TOKEN!;
const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET!;

// Telegram sends all updates here
app.post(`/webhook/${WEBHOOK_SECRET}`, async (req, res) => {
  res.sendStatus(200); // acknowledge immediately — never make Telegram wait

  const update = req.body;
  const message = update?.message || update?.callback_query?.message;
  if (!message?.text) return;

  try {
    const event = parseCommand(message);
    if (!event) {
      await replyToUser(message.chat.id, "❓ Unknown command. Try /help");
      return;
    }

    // Fire and forget to n8n — reply comes back via n8n → bot API
    await dispatchToN8n(event);

    // Immediate acknowledgement to user
    await replyToUser(message.chat.id, "⚡ Processing...", message.message_id);

  } catch (err) {
    console.error("Gateway error:", err);
    await replyToUser(message.chat.id, "⚠️ Something went wrong. Try again.");
  }
});

app.get("/health", (_, res) => res.json({ status: "ok", service: "ifyrt-bot" }));

app.listen(process.env.PORT || 3000, () => {
  console.log("Ifyrt Bot Gateway running");
});
```

## parser.ts — Command → Event

```typescript
import { v4 as uuid } from "uuid";
import { IfyrtEvent } from "./types";

export function parseCommand(message: any): IfyrtEvent | null {
  const text: string = message.text.trim();
  const parts = text.split(/\s+/);
  const command = parts[0].toLowerCase();
  const args = parts.slice(1);

  const base = {
    event_id: uuid(),
    timestamp: new Date().toISOString(),
    source: "telegram" as const,
    user: {
      telegram_id: message.from.id,
      username: message.from.username ?? null,
      first_name: message.from.first_name,
    },
    raw_text: text,
    chat_id: message.chat.id,
    message_id: message.message_id,
  };

  switch (command) {
    case "/start":
      return { ...base, event_type: "user.start", payload: {} };

    case "/backtest":
      return {
        ...base,
        event_type: "backtest.request",
        payload: {
          strategy: args[0] ?? null,
          symbol: args[1] ?? null,
          from: args[2] ?? null,
          to: args[3] ?? null,
          exchange: args[4] ?? "binance",
        },
      };

    case "/simulate":
      return {
        ...base,
        event_type: "simulation.start",
        payload: {
          strategy: args[0] ?? null,
          symbol: args[1] ?? null,
          exchange: args[2] ?? "binance",
        },
      };

    case "/sim_stop":
      return { ...base, event_type: "simulation.stop", payload: {} };

    case "/live_on":
      return { ...base, event_type: "live.enable", payload: {} };

    case "/live_off":
      return { ...base, event_type: "live.disable", payload: {} };

    case "/copy":
      return {
        ...base,
        event_type: "copy.start",
        payload: { target_username: args[0]?.replace("@", "") ?? null },
      };

    case "/copy_stop":
      return { ...base, event_type: "copy.stop", payload: {} };

    case "/status":
      return { ...base, event_type: "user.status", payload: {} };

    case "/subscribe":
      return { ...base, event_type: "subscription.view", payload: {} };

    case "/dashboard":
      return { ...base, event_type: "dashboard.request", payload: {} };

    case "/strategies":
      return { ...base, event_type: "strategy.list", payload: {} };

    case "/setkey":
      // Key is NEVER logged — payload intentionally empty, handled by n8n secure flow
      return { ...base, event_type: "key.submit", payload: {} };

    case "/help":
      return { ...base, event_type: "user.help", payload: {} };

    default:
      return null;
  }
}
```

## dispatcher.ts — Fire to n8n

```typescript
import axios from "axios";
import { IfyrtEvent } from "./types";

const N8N_WEBHOOK_URL = process.env.N8N_GATEWAY_WEBHOOK!;
const N8N_SECRET = process.env.N8N_WEBHOOK_SECRET!;

export async function dispatchToN8n(event: IfyrtEvent): Promise<void> {
  await axios.post(N8N_WEBHOOK_URL, event, {
    headers: {
      "Content-Type": "application/json",
      "X-Ifyrt-Secret": N8N_SECRET,
    },
    timeout: 5000,
  });
}
```

## JWT Design

JWTs are issued by n8n after identity resolution and passed to all Railway workers.

```typescript
// JWT Payload (what gets signed)
interface IfyrtJWTPayload {
  sub: string;          // user UUID (internal)
  telegram_id: number;  // for fast correlation
  tier: "free" | "trial" | "paid";
  live_mode: boolean;
  iat: number;
  exp: number;          // short-lived: 15 minutes
}
```

**Why 15-minute expiry?** These tokens are passed between internal Railway services only — they never leave server-side. Short expiry limits blast radius if one is intercepted. n8n re-issues on each user interaction so users never feel it expire.

**Devil's advocate:** Does JWT even make sense here if all calls are internal service-to-service? You could use a shared secret header (like we already do with `X-Ifyrt-Secret`) everywhere. That's simpler. The counter-argument: JWT carries user context (tier, user_id, live_mode) *inside the token* — workers don't need to hit Supabase to know who they're serving. The statelessness justifies the complexity at scale.

## JWT Issuance — n8n HTTP Function Node

In n8n, after the Supabase user lookup, add a **Function node** that signs the JWT:

```javascript
// n8n Function Node: "Issue JWT"
```

## JWT Verification Middleware (Railway Workers)

Every Railway service (`ifyrt-sim-worker`, `ifyrt-live-exec`, etc.) runs this middleware:

```typescript
// middleware/auth.ts
import jwt from "jsonwebtoken";
import { Request, Response, NextFunction } from "express";

export interface AuthenticatedRequest extends Request {
  user?: {
    sub: string;
    telegram_id: number;
    tier: "free" | "trial" | "paid";
    live_mode: boolean;
  };
}

export function requireAuth(
  req: AuthenticatedRequest,
  res: Response,
  next: NextFunction
) {
  const token = req.headers.authorization?.replace("Bearer ", "");
  if (!token) return res.status(401).json({ error: "No token" });

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET!) as any;
    req.user = payload;
    next();
  } catch {
    return res.status(401).json({ error: "Invalid or expired token" });
  }
}

// Tier-specific guards
export function requirePaid(
  req: AuthenticatedRequest,
  res: Response,
  next: NextFunction
) {
  if (req.user?.tier !== "paid") {
    return res.status(403).json({ error: "Paid subscription required" });
  }
  next();
}

export function requireTrialOrPaid(
  req: AuthenticatedRequest,
  res: Response,
  next: NextFunction
) {
  if (!["trial", "paid"].includes(req.user?.tier ?? "")) {
    return res.status(403).json({ error: "Trial or paid subscription required" });
  }
  next();
}
```

```typescript
import WebSocket from "ws";
import { normalizeSnapshot } from "../normalizer";
import { storeSnapshot } from "../store";

const BINANCE_WS = "wss://stream.binance.com:9443/stream";

export function connectBinance(symbols: string[]) {
  // e.g. ["btcusdt", "ethusdt"]
  const streams = symbols.flatMap(s => [
    `${s}@bookTicker`,
    `${s}@trade`,
  ]).join("/");

  const ws = new WebSocket(`${BINANCE_WS}?streams=${streams}`);

  ws.on("message", async (raw) => {
    const data = JSON.parse(raw.toString());
    const snapshot = normalizeSnapshot("binance", data);
    if (snapshot) await storeSnapshot(snapshot);
  });

  ws.on("close", () => {
    console.warn("Binance WS closed — reconnecting in 3s");
    setTimeout(() => connectBinance(symbols), 3000);
  });

  ws.on("error", (err) => console.error("Binance WS error:", err));
}
```

Auto-reconnect on close — always-on means no gaps in data.

```typescript
export interface Candle {
```

### Simulated Matching Engine

```typescript
// engine/matcher.ts
import { OrderSignal, Position } from "../strategies/interface";
import { MarketSnapshot } from "../types";

export interface Fill {
  filled: boolean;
  fill_price: number;
  fill_qty: number;
  fee: number;
  slippage: number;
}

const FEE_RATE = 0.001; // 0.1% taker fee (Binance standard)

export function simulateFill(
  signal: OrderSignal,
  snapshot: MarketSnapshot,
  latency_ms: number = 50
): Fill {
  if (signal.side === "hold") {
    return { filled: false, fill_price: 0, fill_qty: 0, fee: 0, slippage: 0 };
  }

  const book = signal.side === "buy" ? snapshot.order_book.asks 
                                     : snapshot.order_book.bids;

  let remaining = signal.quantity;
  let total_cost = 0;
  let filled_qty = 0;

  if (signal.type === "market") {
    // Walk the order book — slippage emerges naturally
    for (const [price, qty] of book) {
      if (remaining <= 0) break;
      const take = Math.min(remaining, qty);
      total_cost += take * price;
      filled_qty += take;
      remaining -= take;
    }

    const avg_price = total_cost / filled_qty;
    const ref_price = signal.side === "buy" ? snapshot.ask : snapshot.bid;
    const slippage = Math.abs(avg_price - ref_price);
    const fee = total_cost * FEE_RATE;

    return { filled: true, fill_price: avg_price, fill_qty: filled_qty, fee, slippage };
  }

  if (signal.type === "limit") {
    // Limit fills only if price crosses
    const can_fill = signal.side === "buy"
      ? snapshot.ask <= signal.price!
      : snapshot.bid >= signal.price!;

    if (!can_fill) return { filled: false, fill_price: 0, fill_qty: 0, fee: 0, slippage: 0 };

    const fee = signal.quantity * signal.price! * FEE_RATE * 0.5; // maker fee
    return { filled: true, fill_price: signal.price!, fill_qty: signal.quantity, fee, slippage: 0 };
  }

  return { filled: false, fill_price: 0, fill_qty: 0, fee: 0, slippage: 0 };
}
```

**What makes this high-fidelity:** Market orders walk the real order book depth, so slippage isn't a random number — it emerges from actual liquidity at the moment of execution. Limit orders only fill when price actually crosses. Fees are applied at real exchange rates.

**Devil's advocate:** Does walking the order book actually matter at this scale? For small order sizes (retail users), slippage through the top 5 book levels is usually negligible. The educational value is real though — users learn that large orders move price. Worth keeping.

### Backtest Runner

```typescript
// engine/backtest.ts
import { getOHLCV } from "../data/fetcher";
import { getStrategy } from "../strategies/registry";
import { simulateFill } from "./matcher";
import { calcMetrics } from "./metrics";
import { supabase } from "../db";

export async function runBacktest(job: {
  simulation_id: string;
  user_id: string;
  strategy: string;
  symbol: string;
  exchange: string;
  from: string;
  to: string;
}) {
  const strategy = getStrategy(job.strategy);
  const candles = await getOHLCV(job.exchange, job.symbol, "1h", job.from, job.to);

  let position = { open: false, side: null, entry_price: null, quantity: 0, unrealized_pnl: 0 };
  const trades: any[] = [];
  let balance = 10000; // simulated starting USDT

  for (let i = 50; i < candles.length; i++) {
    const window = candles.slice(0, i + 1);
    const signal = strategy.on_tick(window, position);

    // Fake snapshot from OHLCV for limit/market matching
    const snapshot = candleToSnapshot(candles[i], job.exchange, job.symbol);
    const fill = simulateFill(signal, snapshot);

      trades.push(trade);
```

### Metrics Calculator

```typescript
// engine/metrics.ts
export function calcMetrics(trades: any[], final_balance: number) {
  const initial = 10000;
  const pnl = final_balance - initial;
  const pnl_pct = (pnl / initial) * 100;

  const winning = trades.filter(t => t.pnl > 0);
  const win_rate = trades.length ? (winning.length / trades.length) * 100 : 0;

  // Max drawdown
  let peak = initial, max_drawdown = 0, running = initial;
  for (const t of trades) {
    running += t.pnl;
    if (running > peak) peak = running;
    const dd = (peak - running) / peak * 100;
    if (dd > max_drawdown) max_drawdown = dd;
  }

  // Sharpe (simplified, daily returns assumed)
  const returns = trades.map(t => t.pnl / initial);
  const avg_return = returns.reduce((a, b) => a + b, 0) / (returns.length || 1);
  const std = Math.sqrt(returns.map(r => (r - avg_return) ** 2)
    .reduce((a, b) => a + b, 0) / (returns.length || 1));
  const sharpe = std > 0 ? (avg_return / std) * Math.sqrt(252) : 0;

  return {
    total_trades: trades.length,
    win_rate: +win_rate.toFixed(2),
    pnl,
    pnl_pct: +pnl_pct.toFixed(2),
    max_drawdown: +max_drawdown.toFixed(2),
    sharpe_ratio: +sharpe.toFixed(3),
    final_balance,
  };
}
```

## Key Submission Flow (`/setkey`)

This is the most security-sensitive flow in the entire platform. The key touches exactly two places: the user's Telegram message (immediately deleted) and Supabase Vault.

```
User sends /setkey binance API_KEY API_SECRET
        │
        ▼
ifyrt-bot: parses key + secret from message
  → IMMEDIATELY deletes the Telegram message (bot API deleteMessage)
  → fires key.submit event to n8n (payload: { exchange, key, secret })
        │
        ▼
n8n: receives event
  → stores key+secret into Supabase Vault as secrets
  → stores the vault secret IDs (not the keys) in api_keys table
  → sends Telegram: "🔑 Key saved securely. The message has been deleted."
```

### Bot: Delete the Message Immediately


### Vault Decryption (In-Memory Only)

```typescript
// vault.ts
import { createClient } from "@supabase/supabase-js";

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_KEY!
);

export interface DecryptedKeys {
  api_key: string;
  api_secret: string;
}

export async function getDecryptedKeys(
  user_id: string,
  exchange: string
): Promise<DecryptedKeys> {
  // Fetch vault secret IDs from api_keys table
  const { data: keyRow, error } = await supabase
    .from("api_keys")
    .select("encrypted_key, encrypted_secret")
    .eq("user_id", user_id)
    .eq("exchange", exchange)
    .single();

  if (error || !keyRow) throw new Error("No API key found for user");

  // Decrypt via Supabase Vault — decrypted value lives only in this scope
  const { data: keySecret } = await supabase
    .rpc("vault.decrypted_secrets")
    .eq("id", keyRow.encrypted_key)
    .single();

  const { data: apiSecret } = await supabase
    .rpc("vault.decrypted_secrets")
    .eq("id", keyRow.encrypted_secret)
    .single();

  // Update last_used
  await supabase.from("api_keys")
    .update({ last_used: new Date().toISOString() })
    .eq("user_id", user_id)
    .eq("exchange", exchange);

  return {
    api_key: keySecret.decrypted_secret,
    api_secret: apiSecret.decrypted_secret,
  };
  // Keys go out of scope here — never written to disk, log, or DB
}
```

### Circuit Breaker

This runs on every live fill. If drawdown exceeds threshold, it kills the session and locks the user out of live mode until they manually reset.

```typescript
// circuit-breaker.ts
import { supabase } from "./db";

const MAX_DRAWDOWN_PCT = 10; // 10% session drawdown kills live mode
const MAX_CONSECUTIVE_LOSSES = 5;

export interface CircuitState {
  peak_balance: number;
  current_balance: number;
  consecutive_losses: number;
}

export async function checkCircuit(
  user_id: string,
  state: CircuitState,
  session_id: string
): Promise<{ tripped: boolean; reason?: string }> {
  const drawdown = 
    (state.peak_balance - state.current_balance) / state.peak_balance * 100;

  let reason: string | null = null;

  if (drawdown >= MAX_DRAWDOWN_PCT) {
    reason = `Drawdown limit hit: ${drawdown.toFixed(2)}%`;
  } else if (state.consecutive_losses >= MAX_CONSECUTIVE_LOSSES) {
    reason = `${MAX_CONSECUTIVE_LOSSES} consecutive losses`;
  }

  if (reason) {
    // Kill the session
    await supabase.from("live_sessions").update({
      status: "circuit_broken",
      stopped_at: new Date().toISOString(),
      stop_reason: reason,
    }).eq("id", session_id);

    // Lock user out of live mode
    await supabase.from("users")
      .update({ live_mode: false })
      .eq("id", user_id);

    await supabase.from("circuit_breakers").upsert({
      user_id,
      triggered: true,
      trigger_reason: reason,
      drawdown_pct: drawdown,
      triggered_at: new Date().toISOString(),
    });

    return { tripped: true, reason };
  }

  return { tripped: false };
}
```

When the circuit trips, n8n picks up the `circuit_breakers` row insert via Supabase Realtime and immediately fires this Telegram message:

```
🚨 LIVE MODE SUSPENDED

Reason: Drawdown limit hit: 11.4%

Your live session has been automatically stopped to protect your funds.

No further orders will be placed.

Use /status to review your session.
Use /live_on to re-enable when ready (manual reset required).
```

**Devil's advocate:** A 10% drawdown threshold is arbitrary — for some strategies this is noise, for others it's catastrophic. The right answer is to make it user-configurable at session start. Add a `max_drawdown_pct` column to `live_sessions` and let `/live_on [strategy] [symbol] --stop-loss 5` override the default. Launch with 10% hardcoded, make it configurable in v2.

### Live Runner

```typescript
// runner.ts
import { getDecryptedKeys } from "./vault";
import { BinanceAdapter } from "./adapters/binance";
import { BybitAdapter } from "./adapters/bybit";
import { checkCircuit, CircuitState } from "./circuit-breaker";
import { getStrategy } from "../../shared/strategies/registry";
import { supabase } from "./db";

const activeSessions = new Map<string, { running: boolean }>();

export async function startLiveSession(job: {
  user_id: string;
  session_id: string;
  strategy: string;
  symbol: string;
  exchange: string;
}) {
  const keys = await getDecryptedKeys(job.user_id, job.exchange);
  const adapter = job.exchange === "binance"
    ? new BinanceAdapter(keys.api_key, keys.api_secret)
    : new BybitAdapter(keys.api_key, keys.api_secret);

  // Keys are now referenced only inside adapter closure
  // vault.ts local variables are out of scope

  const strategy = getStrategy(job.strategy);
  const state: CircuitState = {
    peak_balance: 10000, // fetched from exchange in production
    current_balance: 10000,
    consecutive_losses: 0,
  };

  activeSessions.set(job.user_id, { running: true });

  while (activeSessions.get(job.user_id)?.running) {
    // Fetch latest candles
    const { data: candles } = await supabase
      .from("ohlcv")
      .select("*")
      .eq("exchange", job.exchange)
      .eq("symbol", job.symbol)
      .order("open_time", { ascending: false })
      .limit(100);

    const signal = strategy.on_tick(candles!.reverse(), position);
    
    if (signal.side !== "hold") {
      const fill = await adapter.placeOrder(signal, job.symbol);

      // Write real trade
      await supabase.from("trades").insert({
        simulation_id: null,
        user_id: job.user_id,
        is_live: true,        // ← always explicit
        exchange: job.exchange,
        symbol: job.symbol,
        side: signal.side,
        type: signal.type,
        quantity: fill.fill_qty,
        fill_price: fill.fill_price,
        fee: fill.fee,
        slippage: fill.slippage,
        strategy: job.strategy,
      });

      // Update circuit state
      state.current_balance += signal.side === "sell" 
        ? fill.fill_qty * fill.fill_price - fill.fee 
        : -(fill.fill_qty * fill.fill_price + fill.fee);
      if (state.current_balance > state.peak_balance) 
        state.peak_balance = state.current_balance;

      const circuit = await checkCircuit(job.user_id, state, job.session_id);
      if (circuit.tripped) {
        activeSessions.delete(job.user_id);
        break;
      }
    }

    await sleep(60_000); // poll every 1 minute (1h candle strategy)
  }
}

export function stopLiveSession(user_id: string) {
  activeSessions.delete(user_id);
}

const sleep = (ms: number) => new Promise(r => setTimeout(r, ms));
```

# Block C: Copy-Trading Simulation Layer + Dashboard

## Section 19 (Expanded): Copy-Trading Architecture

### Design Philosophy

Copy trading in TriFi/Ifyrt is **signal relay**, not account mirroring. The leader's strategy emits signals. Those signals are published to an internal bus. Followers subscribe. Each follower's signal is processed by their own **isolated simulation worker** (or live runner if they've enabled live mode). The leader never touches a follower's account. The follower never sees the leader's strategy code.

This keeps the architecture clean: copy trading is just a **fan-out layer on top of the existing simulation and live execution engines**. No new execution logic. The copy layer only routes.

---

### Database Schema

```sql
-- Who is allowed to be copied
CREATE TABLE copy_profiles (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL REFERENCES users(id) UNIQUE,
  display_name  TEXT NOT NULL,           -- shown to followers, NOT username
  is_public     BOOLEAN DEFAULT false,   -- discoverable in /copy list
  allow_copy    BOOLEAN DEFAULT true,    -- master kill switch
  created_at    TIMESTAMPTZ DEFAULT now()

-- Active copy relationships
CREATE TABLE copy_subscriptions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  follower_id   UUID NOT NULL REFERENCES users(id),
  leader_id     UUID NOT NULL REFERENCES users(id),
  mode          TEXT NOT NULL CHECK (mode IN ('sim', 'live')),
  -- 'sim'  = follower simulates the copied signals (always allowed)
  -- 'live' = follower routes copied signals to their live runner (paid only)

  created_at    TIMESTAMPTZ DEFAULT now(),

-- Signal events emitted by leaders, fanned out to followers

  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  fill_qty      NUMERIC,

  slippage      NUMERIC,
  executed_at   TIMESTAMPTZ
```

### Signal Emission (Leader Side)

When a leader's strategy runner (sim or live) produces a non-`hold` signal, it calls `emitCopySignal()` **after** its own execution completes. The leader's trade always happens first. Copy is always derivative.

```typescript
// copy-signal-emitter.ts
import { supabase } from "./db";
import { redisPublisher } from "./bus";

export async function emitCopySignal(params: {
  leader_id: string;
  strategy: string;
  symbol: string;
  exchange: string;
  side: "buy" | "sell";
  signal_type: "market" | "limit";
  price?: number;
  quantity: number;
}) {
  // 1. Check leader has copy enabled
  const { data: profile } = await supabase
    .from("copy_profiles")
    .select("allow_copy")
    .eq("user_id", params.leader_id)
    .single();

  if (!profile?.allow_copy) return;

  // 2. Persist signal
  const { data: signal } = await supabase
    .from("copy_signals")
    .insert({
      leader_id:   params.leader_id,
      strategy:    params.strategy,
      symbol:      params.symbol,
      exchange:    params.exchange,
      side:        params.side,
      signal_type: params.signal_type,
      price:       params.price ?? null,
      quantity:    params.quantity,
    })
    .select()
    .single();

  // 3. Fan out to message bus — copy worker picks this up
  await redisPublisher.publish("copy:signals", JSON.stringify({
    signal_id:  signal.id,
    leader_id:  params.leader_id,
  }));
}
```

The signal is intentionally **minimal** — side, type, quantity, symbol. Followers never receive the strategy's internal state, indicator values, or entry reasoning. They see only what a real exchange fill receipt would show.

### Copy Worker (Fan-Out + Dispatch)

A dedicated worker service `ifyrt-copy-worker` subscribes to `copy:signals`. On each signal it:

1. Looks up all active followers of the leader
2. For each follower, checks their mode (`sim` or `live`) and access rights
3. Dispatches to the correct engine
4. Records the execution row

```typescript
// copy-worker.ts
import { redisSubscriber } from "./bus";
import { supabase } from "./db";
import { dispatchSimCopy } from "./dispatch-sim";
import { dispatchLiveCopy } from "./dispatch-live";

redisSubscriber.subscribe("copy:signals", async (message) => {
  const { signal_id, leader_id } = JSON.parse(message);

  // Fetch full signal
  const { data: signal } = await supabase
    .from("copy_signals")
    .select("*")
    .eq("id", signal_id)
    .single();

  if (!signal) return;

  // Fetch all active followers
  const { data: subs } = await supabase
    .from("copy_subscriptions")
    .select("follower_id, mode")
    .eq("leader_id", leader_id)
    .eq("active", true);

  if (!subs?.length) return;

  // Dispatch concurrently — one follower's failure must not block others
  await Promise.allSettled(
    subs.map(sub => dispatchFollower(signal, sub))
  );
});

async function dispatchFollower(
  signal: CopySignal,
  sub: { follower_id: string; mode: "sim" | "live" }
) {
  // Insert pending execution row first (idempotency anchor)
  const { data: exec } = await supabase
    .from("copy_executions")
    .insert({
      signal_id:   signal.id,
      follower_id: sub.follower_id,
      mode:        sub.mode,
      status:      "pending",
    })
    .select()
    .single();

  try {
    if (sub.mode === "sim") {
      await dispatchSimCopy(signal, sub.follower_id, exec.id);
    } else {
      await dispatchLiveCopy(signal, sub.follower_id, exec.id);
    }
  } catch (err) {
    await supabase.from("copy_executions")
      .update({ status: "failed", skip_reason: String(err) })
      .eq("id", exec.id);
  }
}
```

### Sim Copy Dispatch

Routes the copied signal into the follower's simulation engine. Reuses the **exact same simulated matching engine** as native simulation — the only difference is the signal source.

```typescript
// dispatch-sim.ts
import { supabase } from "./db";
import { SimulatedMatchingEngine } from "../../sim-engine/matching-engine";

export async function dispatchSimCopy(
  signal: CopySignal,
  follower_id: string,
  exec_id: string
) {
  const engine = new SimulatedMatchingEngine({
    exchange: signal.exchange,
    symbol:   signal.symbol,
  });

  const fill = await engine.execute({
    side:     signal.side as "buy" | "sell",
    type:     signal.signal_type as "market" | "limit",
    quantity: signal.quantity,
    price:    signal.price,
  });

  // Record as a sim trade — linked to copy_execution, NOT a live session
  await supabase.from("trades").insert({
    user_id:      follower_id,
    is_live:      false,
    simulation_id: null,        // copy trades are not tied to a named simulation
    copy_exec_id: exec_id,      // ← distinguishes copy sim trades from native sim trades
    exchange:     signal.exchange,
    symbol:       signal.symbol,
    side:         signal.side,
    type:         signal.signal_type,
    quantity:     fill.fill_qty,
    fill_price:   fill.fill_price,
    fee:          fill.fee,
    slippage:     fill.slippage,
    strategy:     "copy",
  });

  await supabase.from("copy_executions").update({
    status:      "executed",
    fill_price:  fill.fill_price,
    fill_qty:    fill.fill_qty,
    fee:         fill.fee,
    slippage:    fill.slippage,
    executed_at: new Date().toISOString(),
  }).eq("id", exec_id);
}
```

### Live Copy Dispatch

Routes the copied signal into the follower's **existing live runner**. The follower must have live mode active and a live session open. If not, the execution is marked `skipped` — signals are never queued or retried.

```typescript
// dispatch-live.ts
import { supabase } from "./db";
import { getDecryptedKeys } from "../../live-exec/vault";
import { BinanceAdapter } from "../../live-exec/adapters/binance";
import { BybitAdapter } from "../../live-exec/adapters/bybit";
import { checkCircuit } from "../../live-exec/circuit-breaker";

export async function dispatchLiveCopy(
  signal: CopySignal,
  follower_id: string,
  exec_id: string
) {
  // Gate 1: Paid subscription required for live copy
  const { data: user } = await supabase
    .from("users")
    .select("subscription_status, live_mode")
    .eq("id", follower_id)
    .single();

  if (user?.subscription_status !== "paid") {
    return skip(exec_id, "not_paid");
  }

  // Gate 2: Follower must have live mode enabled
  if (!user?.live_mode) {
    return skip(exec_id, "live_mode_disabled");
  }

  // Gate 3: No active circuit breaker
  const { data: cb } = await supabase
    .from("circuit_breakers")
    .select("triggered")
    .eq("user_id", follower_id)
    .single();

  if (cb?.triggered) {
    return skip(exec_id, "circuit_breaker_active");
  }

  // Gate 4: Active live session exists for this symbol/exchange
  const { data: session } = await supabase
    .from("live_sessions")
    .select("id, exchange")
    .eq("user_id", follower_id)
    .eq("symbol", signal.symbol)
    .eq("status", "active")
    .single();

  if (!session) {
    return skip(exec_id, "no_active_session_for_symbol");
  }

  // All gates passed — execute
  const keys = await getDecryptedKeys(follower_id, session.exchange);
  const adapter = session.exchange === "binance"
    ? new BinanceAdapter(keys.api_key, keys.api_secret)
    : new BybitAdapter(keys.api_key, keys.api_secret);

  const fill = await adapter.placeOrder(
    { side: signal.side, type: signal.signal_type, quantity: signal.quantity, price: signal.price },
    signal.symbol
  );

  await supabase.from("trades").insert({
    user_id:      follower_id,
    is_live:      true,
    copy_exec_id: exec_id,
    exchange:     session.exchange,
    symbol:       signal.symbol,
    side:         signal.side,
    type:         signal.signal_type,
    quantity:     fill.fill_qty,
    fill_price:   fill.fill_price,
    fee:          fill.fee,
    slippage:     fill.slippage,
    strategy:     "copy_live",
  });

  await supabase.from("copy_executions").update({
    status:      "executed",
    fill_price:  fill.fill_price,
    fill_qty:    fill.fill_qty,
    fee:         fill.fee,
    slippage:    fill.slippage,
    executed_at: new Date().toISOString(),
  }).eq("id", exec_id);
}

async function skip(exec_id: string, reason: string) {
  await supabase.from("copy_executions").update({
    status: "skipped",
    skip_reason: reason,
  }).eq("id", exec_id);
}
```

```typescript
// lib/performance.ts
export function computePerformance(trades: Trade[]) {
  const closed = trades.filter(t => t.side === "sell"); // simplified
  const pnls = closed.map(t => 
    (t.fill_price - t.entry_price) * t.fill_qty - t.fee
  );
  const mean = pnls.reduce((a, b) => a + b, 0) / pnls.length;
  const stdDev = Math.sqrt(
    pnls.map(p => Math.pow(p - mean, 2)).reduce((a, b) => a + b, 0) / pnls.length
  );

  return {
    winRate: pnls.filter(p => p > 0).length / pnls.length,
    avgPnl: mean,
    sharpe: stdDev > 0 ? mean / stdDev : 0,
    maxDrawdown: computeMaxDrawdown(pnls),
    totalPnl: pnls.reduce((a, b) => a + b, 0),
  };
}
```

# Block D: Scanner, Scorer & Strategy Configuration Engine

---

## Design Philosophy

The scanner, scorer, and bot configuration system share **one unified Strategy Config object**. The exact same config that defines a backtest defines a live bot defines a sim bot. There is no translation layer, no "convert backtest params to bot params" step. You write the config once and the runtime context (backtest / sim / live) determines how it executes. This is the most important design decision in this block — it's what makes the platform feel coherent rather than bolted together.

```
Market Data
    ↓
Scanner (symbol selection + OHLCV fetch)
    ↓
Scorer (indicator pipeline → weighted score)
    ↓
Signal (score crosses threshold → directional signal)
    ↓
Strategy Config (user-defined rules that gate the signal)
    ↓
Execution Router (backtest engine / sim engine / live engine)
```

## Section 2: The Scanner

The scanner's job is **symbol selection**. It doesn't score — it produces a filtered candidate list that the scorer then processes. Keeping these separate means you can change the liquidity filter without touching scoring logic.

```typescript
// scanner/index.ts
import { supabase } from "../db";
import { fetchTickers } from "../market-data/tickers";
import { fetchOHLCV } from "../market-data/ohlcv";
import { StrategyConfig, OHLCV, ScanResult } from "../types";

export async function runScanner(config: StrategyConfig): Promise<ScanResult[]> {
  // Step 1: Fetch all tickers for the exchange/quote pair
  const tickers = await fetchTickers(config.exchange, config.quote_asset);

  // Step 2: Liquidity pre-filter — eliminates illiquid symbols
  // This is the same filter FancyFang uses to avoid trailing stop rejections
  const liquid = tickers
    .filter(t => t.volume_24h_usdt >= config.min_volume_24h)
    .sort((a, b) => b.volume_24h_usdt - a.volume_24h_usdt)
    .slice(0, config.max_symbols);

  // Step 3: Fetch OHLCV for each candidate (concurrent, rate-limited)
  const results = await Promise.allSettled(
    liquid.map(ticker =>
      fetchOHLCV({
        exchange:  config.exchange,
        symbol:    ticker.symbol,
        timeframe: config.timeframe,
        limit:     config.candles_required,
      })
    )
  );

  // Step 4: Return only successful fetches with enough candle history
  return results
    .map((r, i) => r.status === "fulfilled" ? {
      symbol:  liquid[i].symbol,
      candles: r.value,
      funding: liquid[i].funding_rate ?? 0,
    } : null)
    .filter(Boolean)
    .filter(r => r!.candles.length >= config.candles_required) as ScanResult[];
}
```

## Section 3: The Scorer

The scorer takes a symbol's OHLCV data and produces a single score from **-100 to +100**. Positive = bullish pressure. Negative = bearish pressure. Each indicator contributes a partial score proportional to its weight. This is the exact same model from FancyFang, generalized so users can tune the weights.

### Indicator Modules

Each indicator returns a value from -1.0 to +1.0 (normalized). The scorer multiplies by weight and sums.

```typescript
// scorer/indicators/rsi.ts
export function scoreRSI(candles: OHLCV[], period: number, ob: number, os: number): number {
  const rsi = computeRSI(candles, period);
  const latest = rsi[rsi.length - 1];

  if (latest >= ob) return -1.0;       // overbought → bearish
  if (latest <= os) return 1.0;        // oversold → bullish
  if (latest > 50)  return -((latest - 50) / (ob - 50));   // graded above midpoint
  if (latest < 50)  return (50 - latest) / (50 - os);      // graded below midpoint
  return 0;
}

// scorer/indicators/bollinger.ts
export function scoreBollinger(candles: OHLCV[], period: number, stddev: number): number {
  const { upper, lower, mid } = computeBB(candles, period, stddev);
  const price = candles[candles.length - 1].close;
  const bandwidth = upper - lower;

  if (bandwidth === 0) return 0;

  const position = (price - lower) / bandwidth; // 0 = at lower, 1 = at upper
  // >0.9 = near upper band → bearish, <0.1 = near lower band → bullish
  return 1 - (position * 2); // maps 0→+1, 0.5→0, 1→-1
}

// scorer/indicators/ema.ts
export function scoreEMA(candles: OHLCV[], fast: number, slow: number): number {
  const emaFast = computeEMA(candles, fast);
  const emaSlow = computeEMA(candles, slow);
  const f = emaFast[emaFast.length - 1];
  const s = emaSlow[emaSlow.length - 1];

  const crossPct = (f - s) / s; // normalized distance between EMAs
  return Math.max(-1, Math.min(1, crossPct * 20)); // clamp, amplify small differences
}

// scorer/indicators/volume.ts
export function scoreVolume(candles: OHLCV[], maPeriod: number): number {
  const recent = candles.slice(-maPeriod);
  const ma = recent.reduce((s, c) => s + c.volume, 0) / maPeriod;
  const latest = candles[candles.length - 1].volume;

  const ratio = latest / ma;
  // High volume alone isn't directional — pair with price direction
  const priceDir = candles[candles.length - 1].close > candles[candles.length - 2].close ? 1 : -1;
  return Math.max(-1, Math.min(1, (ratio - 1) * priceDir));
}

// scorer/indicators/funding.ts
export function scoreFunding(fundingRate: number): number {
  // Positive funding → longs pay shorts → bearish lean
  // Negative funding → shorts pay longs → bullish lean
  const clamped = Math.max(-0.002, Math.min(0.002, fundingRate)); // cap at ±0.2%
  return -(clamped / 0.002); // normalize to -1..+1, inverted
}

// scorer/indicators/patterns.ts
export function scorePatterns(candles: OHLCV[]): number {
  let score = 0;
  const signals: number[] = [];

  // Hammer (bullish)
  if (isHammer(candles))        signals.push(1.0);
  // Engulfing bullish
  if (isBullEngulf(candles))    signals.push(1.0);
  // Morning star
  if (isMorningStar(candles))   signals.push(0.8);
  // Shooting star (bearish)
  if (isShootingStar(candles))  signals.push(-1.0);
  // Engulfing bearish
  if (isBearEngulf(candles))    signals.push(-1.0);
  // Evening star
  if (isEveningStar(candles))   signals.push(-0.8);
  // Doji (indecision — weakens any signal)
  if (isDoji(candles))          signals.push(0);

  if (signals.length === 0) return 0;
  return signals.reduce((a, b) => a + b, 0) / signals.length;
}
```

### Master Scorer

```typescript
// scorer/index.ts
import { scoreRSI } from "./indicators/rsi";
import { scoreBollinger } from "./indicators/bollinger";
import { scoreEMA } from "./indicators/ema";
import { scoreVolume } from "./indicators/volume";
import { scoreFunding } from "./indicators/funding";
import { scorePatterns } from "./indicators/patterns";
import { StrategyConfig, ScanResult, ScoredSymbol } from "../types";

export function scoreSymbol(result: ScanResult, config: StrategyConfig): ScoredSymbol {
  const { candles, symbol, funding } = result;

  const components = {
    rsi:      scoreRSI(candles, config.rsi_period, config.rsi_ob, config.rsi_os),
    bb:       scoreBollinger(candles, config.bb_period, config.bb_stddev),
    ema:      scoreEMA(candles, config.ema_fast, config.ema_slow),
    volume:   scoreVolume(candles, config.volume_ma_period),
    funding:  scoreFunding(funding),
    patterns: scorePatterns(candles),
  };

  // Weighted sum, scaled to -100..+100
  const totalWeight = (
    config.weight_rsi + config.weight_bb + config.weight_ema +
    config.weight_volume + config.weight_funding + config.weight_patterns
  );

  const raw =
    components.rsi      * config.weight_rsi      +
    components.bb       * config.weight_bb        +
    components.ema      * config.weight_ema       +
    components.volume   * config.weight_volume    +
    components.funding  * config.weight_funding   +
    components.patterns * config.weight_patterns;

  const score = Math.round((raw / totalWeight) * 100);

  return {
    symbol,
    score,
    components,   // stored for explainability — users can see why a symbol scored
    candles,
    funding,
    scored_at: new Date().toISOString(),
  };
}

export function runScorer(results: ScanResult[], config: StrategyConfig): ScoredSymbol[] {
  return results
    .map(r => scoreSymbol(r, config))
    .sort((a, b) => Math.abs(b.score) - Math.abs(a.score)); // strongest signals first
}
```

## Section 4: Signal Generation

The scorer produces numbers. The signal layer converts numbers into actionable trade instructions by applying the strategy config's threshold rules.

```typescript
// signal/index.ts
import { ScoredSymbol, StrategyConfig, Signal } from "../types";

export function generateSignals(
  scored: ScoredSymbol[],
  config: StrategyConfig,
  openPositions: Set<string>  // symbols already in a position
): Signal[] {
  const signals: Signal[] = [];

  for (const s of scored) {
    // Direction filter
    const canLong  = config.direction !== "short";
    const canShort = config.direction !== "long";

    // Entry signals
    if (canLong && s.score >= config.score_min_long && !openPositions.has(s.symbol)) {
      signals.push({
        symbol:    s.symbol,
        side:      "buy",
        type:      "market",
        quantity:  deriveQuantity(config),
        score:     s.score,
        components: s.components,
        reason:    `Score ${s.score} ≥ long threshold ${config.score_min_long}`,
      });
    }

    if (canShort && s.score <= config.score_max_short && !openPositions.has(s.symbol)) {
      signals.push({
        symbol:    s.symbol,
        side:      "sell",
        type:      "market",
        quantity:  deriveQuantity(config),
        score:     s.score,
        components: s.components,
        reason:    `Score ${s.score} ≤ short threshold ${config.score_max_short}`,
      });
    }

    // Exit signals (for open positions)
    if (openPositions.has(s.symbol)) {
      if (s.score <= config.score_exit_long)  {
        signals.push({ symbol: s.symbol, side: "close_long",  score: s.score, reason: "Exit: score fell below long exit threshold" });
      }
      if (s.score >= config.score_exit_short) {
        signals.push({ symbol: s.symbol, side: "close_short", score: s.score, reason: "Exit: score rose above short exit threshold" });
      }
    }
  }

  // Respect max concurrent positions
  const entrySignals = signals.filter(s => s.side === "buy" || s.side === "sell");
  const exitSignals  = signals.filter(s => s.side === "close_long" || s.side === "close_short");
  const slotsAvailable = config.max_positions - openPositions.size;

  return [
    ...exitSignals,
    ...entrySignals.slice(0, slotsAvailable),
  ];
}

function deriveQuantity(config: StrategyConfig): number {
  // For perps: quantity = (margin * leverage) / entry_price
  // Entry price unknown at signal time — resolved at execution
  // Return notional margin for now; execution layer handles conversion
  return config.margin_usdt;
}
```

## Section 5: The Scan Loop

This ties scanner + scorer + signal together into a running cycle. Same loop runs in sim mode and live mode — the only difference is what happens to signals at the end.

```typescript
// scan-loop.ts
import { runScanner } from "./scanner";
import { runScorer } from "./scorer";
import { generateSignals } from "./signal";
import { supabase } from "./db";
import { redisPublisher } from "./bus";

export async function startScanLoop(session_id: string) {
  const { data: session } = await supabase
    .from("bot_sessions")
    .select("*, strategies(*)")
    .eq("id", session_id)
    .single();

  const config = session.strategies;
  const positionTracker = new Map<string, number>(); // symbol → cooldown candles remaining
  const openPositions   = new Set<string>();

  while (true) {
    const { data: live } = await supabase
      .from("bot_sessions")
      .select("status")
      .eq("id", session_id)
      .single();

    if (live?.status !== "running") break;

    // --- Scan ---
    const scanResults = await runScanner(config);

    // --- Score ---
    const scored = runScorer(scanResults, config);

    // --- Persist scores ---
    await supabase.from("scan_scores").insert(
      scored.map(s => ({
        session_id,
        strategy_id: config.id,
        symbol:      s.symbol,
        score:       s.score,
        components:  s.components,
        timeframe:   config.timeframe,
        candle_open: s.candles[s.candles.length - 1].open_time,
      }))
    );

    // --- Cooldown tick ---
    for (const [sym, remaining] of positionTracker) {
      if (remaining <= 1) positionTracker.delete(sym);
      else positionTracker.set(sym, remaining - 1);
    }

    // --- Filter out symbols in cooldown ---
    const eligible = scored.filter(s => !positionTracker.has(s.symbol));

    // --- Generate signals ---
    const signals = generateSignals(eligible, config, openPositions);

    // --- Publish signals to execution bus ---
    for (const signal of signals) {
      await redisPublisher.publish(`signals:${session.mode}:${session.user_id}`, JSON.stringify({
        session_id,
        user_id:  session.user_id,
        mode:     session.mode,
        signal,
      }));

      // Track cooldowns on entry
      if (signal.side === "buy" || signal.side === "sell") {
        openPositions.add(signal.symbol);
        positionTracker.set(signal.symbol, config.cooldown_candles);
      }
      if (signal.side === "close_long" || signal.side === "close_short") {
        openPositions.delete(signal.symbol);
        positionTracker.set(signal.symbol, config.cooldown_candles);
      }
    }

    await supabase.from("bot_sessions")
      .update({ last_scan_at: new Date().toISOString() })
      .eq("id", session_id);

    // Wait for next candle close (approximate)
    await sleep(session.scan_interval_s * 1000);
  }
}
```

## Section 6: Backtesting

The backtest engine uses the **exact same scorer, signal generator, and strategy config**. The only difference is data source: instead of fetching live market data, it iterates over historical OHLCV rows candle by candle in chronological order. This is what makes backtesting trustworthy — the code path is not separate.

```typescript
// backtest/engine.ts
import { runScorer } from "../scorer";
import { generateSignals } from "../signal";
import { SimulatedMatchingEngine } from "../sim-engine/matching-engine";
import { supabase } from "../db";

export async function runBacktest(session_id: string) {
  const { data: session } = await supabase
    .from("bot_sessions")
    .select("*, strategies(*)")
    .eq("id", session_id)
    .single();

  const config = session.strategies;

  // Fetch historical OHLCV for all symbols in the universe
  // Pre-filtered by liquidity (same filter as live scanner, applied at backtest start)
  const { data: symbols } = await supabase
    .from("ohlcv_symbols")
    .select("symbol")
    .eq("exchange", config.exchange)
    .gte("avg_volume_30d", config.min_volume_24h);

  const { data: allCandles } = await supabase
    .from("ohlcv")
    .select("*")
    .in("symbol", symbols!.map(s => s.symbol))
    .eq("exchange", config.exchange)
    .eq("timeframe", config.timeframe)
    .gte("open_time", session.backtest_start)
    .lte("open_time", session.backtest_end)
    .order("open_time", { ascending: true });

  // Group candles by symbol
  const bySymbol = groupBy(allCandles!, "symbol");

  // Build timeline: sorted unique candle timestamps
  const timeline = [...new Set(allCandles!.map(c => c.open_time))].sort();

  const matchingEngine = new SimulatedMatchingEngine({ config });
  const openPositions  = new Set<string>();
  const cooldowns      = new Map<string, number>();
  const equityCurve: { time: string; equity: number }[] = [];
  let equity = config.margin_usdt * config.max_positions; // starting capital

  for (const timestamp of timeline) {
    // Build scan result for this candle close: for each symbol, take all
    // candles up to and including this timestamp
    const scanResults = Object.entries(bySymbol)
      .map(([symbol, candles]) => {
        const window = candles.filter(c => c.open_time <= timestamp);
        if (window.length < config.candles_required) return null;
        return {
          symbol,
          candles: window.slice(-config.candles_required),
          funding: window[window.length - 1].funding_rate ?? 0,
        };
      })
      .filter(Boolean);

    // Score
    const scored = runScorer(scanResults as any, config);

    // Cooldown tick
    for (const [sym, rem] of cooldowns) {
      if (rem <= 1) cooldowns.delete(sym);
      else cooldowns.set(sym, rem - 1);
    }

    const eligible = scored.filter(s => !cooldowns.has(s.symbol));
    const signals  = generateSignals(eligible, config, openPositions);

    for (const signal of signals) {
      const candles = bySymbol[signal.symbol];
      const currentCandle = candles.find(c => c.open_time === timestamp)!;

      const fill = await matchingEngine.execute({
        signal,
        candle: currentCandle,
        timestamp,
      });

      // Record backtest trade
      await supabase.from("trades").insert({
        user_id:      session.user_id,
        session_id,
        is_live:      false,
        is_backtest:  true,
        exchange:     config.exchange,
        symbol:       signal.symbol,
        side:         signal.side,
        type:         signal.type,
        quantity:     fill.fill_qty,
        fill_price:   fill.fill_price,
        fee:          fill.fee,
        slippage:     fill.slippage,
        strategy:     config.name,
        score:        signal.score,
        candle_time:  timestamp,
      });

      // Update equity
      equity += fill.pnl ?? 0;

      // Update position tracking
      if (signal.side === "buy" || signal.side === "sell") {
        openPositions.add(signal.symbol);
        cooldowns.set(signal.symbol, config.cooldown_candles);
      }
      if (signal.side === "close_long" || signal.side === "close_short") {
        openPositions.delete(signal.symbol);
        cooldowns.set(signal.symbol, config.cooldown_candles);
      }
    }

    equityCurve.push({ time: timestamp, equity });
  }

  // Store results summary
  const trades = await supabase.from("trades")
    .select("*").eq("session_id", session_id);

  await supabase.from("bot_sessions").update({
    status: "completed",
    stopped_at: new Date().toISOString(),
  }).eq("id", session_id);

  await supabase.from("backtest_results").insert({
    session_id,
    strategy_id:   config.id,
    equity_curve:  equityCurve,
    final_equity:  equity,
    total_trades:  trades.data?.length ?? 0,
    ...computePerformance(trades.data ?? []),
  });
}
```

## Payment Portal Architecture

The payment flow has two entry points: the landing page and the Telegram bot itself (via `/subscribe`). Both converge on Stripe and feed back into the same Supabase subscription record.

---

### Stripe Product Setup

```
Stripe Dashboard Configuration:

Product:    Ifyrt Subscription
Price:      $6.99 / month (recurring)
Trial:      7 days (Stripe-native free trial)
Currency:   USD

Metadata on Stripe Customer:
  telegram_id: {user's Telegram ID}
  ifyrt_user_id: {Supabase users.id}
```

Two ways to create a Stripe Checkout session:

**Path A — Landing page (new user, no Telegram session yet):**  
User clicks CTA → enters Telegram username → Checkout session created → on success, bot sends them a `/start` message automatically.

**Path B — Telegram bot (`/subscribe` command):**  
n8n creates a Stripe Checkout session with `telegram_id` pre-filled as metadata → sends user a short-lived payment link in Telegram → user pays → webhook fires → subscription activated.

```
}
```

### Webhook Receiver Service (`ifyrt-payments`)

New Railway service. Listens for Stripe webhook events and updates Supabase accordingly.

**Service responsibilities:**
```
✅ Receive POST from Stripe (signed with STRIPE_WEBHOOK_SECRET)
✅ Verify signature before processing
✅ Update Supabase subscriptions table
✅ Notify user in Telegram on status change
✅ Log all events to audit_log
```

**Events to handle:**

| Stripe Event | Action |
|---|---|
| `checkout.session.completed` | Set `tier = 'paid'`, `trial_start = now()`, `trial_end = now() + 7d`, `stripe_customer_id`, `stripe_subscription_id` |
| `customer.subscription.updated` | Update `paid_until` to new period end |
| `customer.subscription.deleted` | Set `tier = 'free'`, clear `paid_until` — send Telegram notice |
| `invoice.payment_failed` | Send Telegram warning: "Payment failed — update card to keep access" |
| `invoice.payment_succeeded` | Update `paid_until`, optionally send Telegram confirmation |

**Webhook handler pseudocode:**

```typescript
// ifyrt-payments/src/index.ts

app.post("/stripe/webhook", express.raw({ type: "application/json" }), async (req, res) => {
  const sig = req.headers["stripe-signature"]!;
  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(req.body, sig, process.env.STRIPE_WEBHOOK_SECRET!);
  } catch (err) {
    return res.status(400).send("Webhook signature verification failed");
  }

  res.sendStatus(200); // acknowledge immediately

  const { telegram_id, ifyrt_user_id } = extractMetadata(event);

  switch (event.type) {

    case "checkout.session.completed": {
      const session = event.data.object as Stripe.Checkout.Session;
      await supabase.from("subscriptions").update({
        tier: "paid",
        trial_start: new Date().toISOString(),
        trial_end: addDays(new Date(), 7).toISOString(),
        stripe_customer_id: session.customer as string,
        stripe_subscription_id: session.subscription as string,
      }).eq("user_id", ifyrt_user_id);

      await sendTelegram(telegram_id,
        "✅ Trial started! You have full access for 7 days.\n\n" +
        "Use /strategies to get started, or /help to see all commands."
      );
      break;
    }

    case "customer.subscription.deleted": {
      await supabase.from("subscriptions").update({
        tier: "free",
        paid_until: null,
      }).eq("user_id", ifyrt_user_id);

      await sendTelegram(telegram_id,
        "⛔ Your Ifyrt subscription has ended.\n\n" +
        "Backtesting remains free. Use /subscribe to reactivate full access."
      );
      break;
    }

    case "invoice.payment_failed": {
      await sendTelegram(telegram_id,
        "⚠️ Payment failed for your Ifyrt subscription.\n\n" +
        "Update your payment method to keep access: {stripe_portal_url}"
      );
      break;
    }

    case "invoice.payment_succeeded": {
      const invoice = event.data.object as Stripe.Invoice;
      const periodEnd = new Date((invoice.lines.data[0].period.end) * 1000);
      await supabase.from("subscriptions").update({
        paid_until: periodEnd.toISOString(),
      }).eq("user_id", ifyrt_user_id);
      break;
    }
  }

  await supabase.from("audit_log").insert({
    event_type: `stripe.${event.type}`,
    user_id: ifyrt_user_id,
    metadata: { stripe_event_id: event.id },
  });
});
```

### Supabase Schema — Subscriptions Table (Updated)

```sql
ALTER TABLE subscriptions ADD COLUMN IF NOT EXISTS stripe_customer_id     TEXT UNIQUE;
ALTER TABLE subscriptions ADD COLUMN IF NOT EXISTS stripe_subscription_id TEXT UNIQUE;
ALTER TABLE subscriptions ADD COLUMN IF NOT EXISTS paid_until             TIMESTAMPTZ;

-- Full updated subscriptions table:
CREATE TABLE subscriptions (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id               UUID NOT NULL REFERENCES users(id) UNIQUE,
  tier                  TEXT NOT NULL DEFAULT 'free'
                          CHECK (tier IN ('free', 'trial', 'paid')),
  trial_start           TIMESTAMPTZ,
  trial_end             TIMESTAMPTZ,
  paid_until            TIMESTAMPTZ,
  stripe_customer_id    TEXT UNIQUE,
  stripe_subscription_id TEXT UNIQUE,
  created_at            TIMESTAMPTZ DEFAULT now(),
  updated_at            TIMESTAMPTZ DEFAULT now()
);
```

## Design Notes

**Visual identity:** Dark background (deep navy #0A0F1E or near-black), accent color should evoke precision — electric blue (#3B82F6) or cool teal (#14B8A6) works well against dark. Monospace font for any code/terminal elements (e.g. `JetBrains Mono`). Clean sans-serif for body copy (e.g. `Inter`).

**Tone:** Intelligent, calm, educational. Avoid phrases like "moon," "alpha," "10x your portfolio." The platform's differentiator is fidelity and safety — lean into that. Users trust Ifyrt because it doesn't oversell.

**Key conversion logic:** The landing page does not need to explain the full system — it needs to make one action obvious: open the bot. Every section should be oriented toward that CTA. The pricing section is the only place where Stripe enters the picture for most users; the trial starts free, in Telegram, and the payment moment arrives naturally via the bot after 7 days.
