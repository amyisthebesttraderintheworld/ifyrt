# Foundational Archive

These are the original planning and specification documents that shaped the first implementation pass. They have been renamed for clarity and archived so the repo root stays focused on active product work.

## Archived documents

- `ifyrt-clean-design-spec.md`: the clearest architecture-oriented product spec and the main baseline used for the current build.
- `ifyrt-complete-design-spec.txt`: the long-form expanded design document with deeper implementation detail and product copy direction.
- `ifyrt-enhanced-production-spec.txt`: the concise production-focused spec that restores the key system philosophy and constraints.

## Why archived

- They remain valuable as source material and historical context.
- They are no longer the best place for active implementation entry, now that the repo has code, contracts, schema, and service docs.
